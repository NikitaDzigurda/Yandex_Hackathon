## ТЕХНОЛОГИЧЕСКИЙ СТЕК (строго соблюдать)

ИИ-агенты: настраиваются через Yandex Cloud — доступны:

  - Qwen3 235B, DeepSeek 3.2, GPT OSS 120B
  - Инструменты агентов подключаются через MCP

Бэкенд: Python + FastAPI

для проверки логики — допускается простой HTML или Streamlit. Полноценный UI — отдельная фаза.

Развёртывание: всё должно подниматься через docker-compose на чистом окружении.

## АРХИТЕКТУРА АГЕНТОВ (уже спроектирована, реализуем поэтапно)

Система состоит из 7 агентов под управлением Orchestrator:

1. Intake Agent     — приём и оценка заявок по 5 критериям, диалог с инициатором
2. Research Agent   — глубокий ресёрч источников, выдвижение гипотез, отчёт
3. Planning Agent   — роадмап, компетенции, подбор команды
4. DevOps Agent     — автоматизация запросов на ресурсы Yandex Cloud
5. Monitor Agent    — мониторинг Трекера и Sourcecraft, выявление зависших задач
6. Reporting Agent  — черновики отчётов, описания результатов, публикации
7. Orchestrator     — координация всех агентов, состояние проекта, эскалация

MVP фокус (в первую очередь): Intake Agent + Research Agent + Orchestrator

## ЭТАПЫ ЖИЗНЕННОГО ЦИКЛА ПРОЕКТА

1. Приём и первичная оценка заявок
   Сейчас: оценка по 5 критериям вручную, досбор информации от инициатора
   Агент: оценивает по критериям, задаёт уточняющие вопросы, формирует резюме для РП

2. Глубокий ресёрч
   Сейчас: ручной поиск источников, статей, формулировка гипотез
   Агент: поиск и анализ источников, ранжирование гипотез, структурированный отчёт

3. Планирование и поиск ресурсов
   Сейчас: роадмап, описание задач, требования к компетенциям, поиск команды
   Агент: генерация документов роадмапа, описания компетенций, ранжирование кандидатов

4. Выделение инфраструктуры и доступов
   Сейчас: ручное нарезание ресурсов в Yandex Cloud, выдача доступов команде
   Агент: автоматизация запросов, шаблоны конфигураций, выдача/отзыв доступов

5. Реализация проекта
   Сейчас: отслеживание коммуникаций, прогресса, изменений в репозиториях
   Агент: мониторинг репозиториев и трекера, сводки по прогрессу, выявление рисков

6. Подготовка к завершению
   Сейчас: ручная подготовка материалов для публикации
   Агент: черновики отчётов, описание результатов, структура публикаций

7. Завершение и масштабирование
   Сейчас: анонсы, публикации, роспуск команды
   Агент: подготовка анонсов, сбор метрик, рекомендации по масштабированию

## HUMAN-IN-THE-LOOP: где человек обязателен

Следующие действия агент НЕ выполняет без явного апрува человека (РП):
- Финальное одобрение или отклонение заявки
- Запрос на выделение реальных ресурсов Yandex Cloud
- Публикация итоговых документов
- Роспуск команды проекта

Везде остальное агент выполняет автономно и информирует РП через Трекер.
