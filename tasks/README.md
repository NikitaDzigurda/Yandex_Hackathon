# Задания хакатона

В этой папке лежат **копии официальных формулировок кейсов** от организаторов. Команда выбирает **одно** задание (1, 2 или 3); открытые критерии оценки — внутри соответствующего файла (до **100 баллов** на кейс).

Решение вы оформляете в корне репозитория и в каталогах `docs/` и `src/` согласно регламенту организаторов.

## Список заданий (нумерация 1 / 2 / 3)


| №   | Файл                                                       | Кратко о кейсе                                                                                                                              |
| --- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | [task-01-sirius.md](task-01-sirius.md)                     | Внутривузовская платформа для распределения задач между сотрудниками и студентами (MVP, роли, доска задач, баллы и репутация).              |
| 2   | [task-02-yandex-shad.md](task-02-yandex-shad.md)           | Reciper — мобильный AI-ассистент питания, рецептов и готовки (MVP приложения и инфраструктуры).                                             |
| 3   | [task-03-yandex-cloud-cto.md](task-03-yandex-cloud-cto.md) | Платформа автоматизации управления проектной деятельностью на базе ИИ-агентов (концепция и/или прототип, интеграции с экосистемой Яндекса). |


## Где описывать решение

- Укажите выбранное задание **явно** в корневом `README.md` (и при необходимости в `docs/`).
- Документация по решению — в `docs/`, код — в `src/`.
- Воспроизводимый запуск и проверки — в `DEPLOY.md` и через `docker compose` по `docker-compose.yml` (требования организаторов).

Детальные критерии оценки — только в файле выбранного кейса (`task-01-…`, `task-02-…` или `task-03-…`).
