# [Название команды] - Хакатон по вайбкодингу

Добро пожаловать в репозиторий команды **[Название команды]**!

## О репозитории
Этот репозиторий создан для участия в хакатоне по вайбкодингу. Здесь вы публикуете финальное решение выбранного задания в стандартизированном виде, чтобы агент-оценщик смог воспроизводимо развернуть и протестировать проект.

## Структура репозитория

```
team-repo/
├── README.md                 # Обзор решения + какое задание выбрано
├── DEPLOY.md                 # Только воспроизводимые команды для развертывания/тестов
├── docker-compose.yml        # Обязательный docker compose файл
├── tasks/                    # Копии всех заданий (для справки)
├── docs/                     # Документация по вашему решению (бизнес-часть)
└── src/                      # Код решения (код-часть)
```

Необходимо соблюдать структуру проекта, файлы которые нужно заполнить нужно заполнить. Этот readme можно менять в произвольном порядке.

## Обязательные требования

1. **Выбранное задание должно быть явно указано**
   - В этом `README.md` укажите, какое задание вы выбрали.
   - В `docs/` опишите постановку и бизнес-логику именно выбранного задания.

2. **Все решение должно подниматься через docker compose**
   - В корне репозитория должен быть `docker-compose.yml`.
   - Агент-оценщик разворачивает и тестирует решение только по `DEPLOY.md`.

3. **`DEPLOY.md` — только воспроизводимые инструкции**
   - Без «вручную откройте IDE» и без неявных шагов.
   - Команды запуска/тестов/проверок должны работать на чистом окружении с Docker.

## Как начать (для команды)

1. **Изучите документацию**
   - Прочитайте `docs/team-guide.md` - полное руководство для команды
   - Ознакомьтесь с `docs/submission-guide.md` - как сдавать решения

2. **Познакомьтесь с заданиями**
   - Посмотрите `tasks/` - доступные задания
   - Выберите одно задание (1/2/3) и зафиксируйте выбор в `README.md`

3. **Начните работать**
   - Пишите документацию по решению в `docs/`
   - Пишите код решения в `src/`
   - Добавьте `docker-compose.yml`
   - Опишите воспроизводимый запуск и тесты в `DEPLOY.md`

## Быстрый старт

```bash
# 1. Клонирование репозитория
git clone https://git.sourcecraft.dev/urfu/[repo]
cd [repo]

# 2. Запуск и проверка (смотрите DEPLOY.md)
# Важно: решение должно подниматься через docker compose.

# 3. Создание коммита с решением
git add .
git commit -m "Финальная версия решения: [краткое описание]"
git push
```
